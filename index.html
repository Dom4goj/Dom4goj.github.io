WhatsApp Redesign

I often use the WhatsApp application for catching up with my friends and family. While using it, I realised there are some elements that are bad UX.

So I decided to redesign it.

  • Nav bar repositioned to the bottom of the screen for easier navigation
  • Increased font sizes for better readability
  • Repainted gray icons and text into light green to match the WhatsApp brand
  • Repositioned timestamps to the midle so users can have easier time distinguishing which messages are theirs
  • Remade input fields on the profile screen for better readability

Landing page

For practice I designed a landing page for an imaginary company that sells and delivers Neon signs. While considering user's needs and thoughts, I made a landing page with the focus on the user.

  • Flashy "Order now" CTA because ordering the product is the most important thing for the user
  • Big heading to tell the user what they are ordering
  • Free delivery on a visible spot to give the user a positive emotion because that makes users buy the product more
  • Rest of the landing page is focused on the quality of product and its production because when users know more about the product they are more likely to buy it as well

Cart/payment screens

I designed an application for a company that delivers craft beer to buyers as a project for my UX/UI Designer course. Here is a payment process from that project.

  • Labeling steps on which the user is currently on the screen is important for better navigation
  • Users need to be able to change their order while on the "Cart" screen for better accessibility
  • It's important to always show the user their final price and divide the costs so they know what they are paying for, because that makes customers trust your company more
  • Input forms need to be properly grouped for easier processing of information

Pricing plans

I designed a simple pricing page for a hiking assistant application. Here are some important elements that should be thought about when creating any pricing plan pages.

  • Heading is your final push to make the user buy the product, so put something encouraging
  • Price is the most important thing to the user, so make it BIG
  • Like with any other element of your application or website, there should be only 1 main component and with the pricing plan, which is usually the "popular plan"
  • Users trust companies more if other users give good reviews, so show them to the user
  • Pricing plans can include a lot of things, so it's always better to find some intuitive way to give them all the information, like putting in the more expensive plan something similar to "Everything in the other plans plus..."
