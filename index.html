Projects

WhatsApp Redesign

I often use the WhatsApp application for catching up with my friends and family. While using it, I started thinking about the UX of the application and realised there are some errors that are contradictory to modern UX rules.

So I decided to redesign the elements that are wrong according to my UX knowledge. The navigation bar is on the top of the screen, so I put it at the bottom for users to easily navigate the application

All the font sizes were too small for users that are older or have bad vision. So I made it bigger for better readability. I changed some gray icons and text into light green that matches the WhatsApp brand.

I repositioned the timestamps to the middle of the screen in the text bubbles so users can have an easier time recognizing which messages are theirs and which are the other people. I changed some input fields into to the usual input boxes because at first I didn’t realise they were input fields in the usual application.

Landing page

For practice I designed a landing page for an imaginary company that sells and delivers Neon signs. While considering user's needs and thoughts, I made a landing page with the focus on the user.

The most important thing to the user is ordering the product, so the CTA with the label “Order now” is in the middle of the screen, emphasized with the red color that contrasts with the black around it.

Next to the CTA is the header that tells the user what they are ordering and that the delivery is free so that the user instantly thinks “Hey, that's nice”. Users that have positive emotions during the contact with the company are more likely to buy the product.

The focus of the website are personalised neon signs, so on the rest of the landing page is information about high quality materials and the process of making the signs. Users that know more about the product are more likely to buy it as well.

Payment process

I designed an application for a company that delivers craft beer to buyers as a project for my UX/UI Designer course. Here is a payment process from that project.

On the top bar is a label that tells the user where they currently are in the application. Users will always easily remember which application they are using right now, so there is no need for a logo on the top of every screen. Users prefer to know at which step in the ordering process they are currently at.

While on the Cart screen, the user can still change their order. Some users might add more products to the cart that way.

It is important to show the users the total amount of their payment, including tax and delivery costs. Adding the tax and delivery costs to the total in the later steps of the payment process makes the user think you want to trick them. Users buy more products from companies that they can trust.

It is important for the input forms to be properly grouped in categories like personal information, user location, card information,... Grouping elements makes it easier for users to process the information.
